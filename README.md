# 武侠 · 姜云行金庸作品插画集 — PDF Extractor

Extract paired text + illustration content from a Chinese wuxia comic PDF (`data/jiang-yun-xing.pdf`). Each PDF page has Traditional Chinese text on the left and a comic illustration on the right — this pipeline splits them apart, OCRs the text, translates it to Simplified Chinese, and organizes everything into chapter-named folders.

## Pipeline

```
PDF (pages 6-212)
    │
    ▼ [PyMuPDF]
Page JPEGs (300 DPI)
    │
    ▼ [Pillow]
Left crop (text)          Right crop (illustration)
    │                           │
    ▼ [PaddleOCR GPU]           ▼ [Real-ESRGAN optional]
Traditional Chinese        Upscaled illustration (PNG)
    │
    ▼ [OpenCC]
Simplified Chinese
    │
    ▼ [chapter title regex]
wuxia/第九回_鐵槍破犁/
    ├── text.txt
    └── illustration.png
```

## Quick Start

### Phase 1 — Extract & Split (Local, ~2 min) ✓ Done

```bash
python main_local.py
```

Already run — outputs ready in `crops_left.zip` and `crops_right.zip`.

### Phase 2 — OCR (AutoDL GPU)

Upload the zip files to an [AutoDL](https://autodl.com) GPU instance:

```bash
# On local machine
scp -P 52290 crops_left.zip crops_right.zip \
  main_autodl.py ocr_text.py config.py upscale.py \
  root@connect.westb.seetacloud.com:/root/

# On AutoDL instance
unzip crops_left.zip -d /root/wuxia_crops/
unzip crops_right.zip -d /root/wuxia_crops/
pip install paddlepaddle-gpu paddleocr pillow
python main_autodl.py
```

Estimated cost: ¥1-2 (~$0.15-0.30) on an RTX 4090 instance.

### Phase 3 — Assemble (Local, ~1 min)

```bash
# Download AutoDL results
scp -P 52290 -r root@connect.westb.seetacloud.com:/root/wuxia_output/ ./tmp_results/

# Run final assembly
python main_local.py --phase3
```

Output lands in `wuxia/` — each chapter as a folder with `text.txt` + `illustration.png`.

## Key Design Decisions

| Decision | Rationale |
|---|---|
| **Hybrid local + cloud** | PaddleOCR (best Traditional Chinese OCR) lacks Apple Silicon support. Local for CPU work, AutoDL GPU for OCR. |
| **300 DPI rasterisation** | Native PDF is 72 PPI — upscaling to 300 DPI gives ~5500×3750px for better OCR and illustration quality. |
| **PNG for illustrations** | Lossless format avoids generational quality loss when image-to-video tools re-encode. |
| **OpenCC for translation** | Fast, offline, deterministic character mapping — no API calls needed. |
| **Chapter title folder names** | `第九回_鐵槍破犁` instead of numeric page numbers — human-readable. Falls back to `page_NNN` for mid-chapter pages. |

## Optional Enhancements

- **Auto-upscale:** Set `UPSCALE_ENABLED = True` in `config.py` — Real-ESRGAN will upscale illustrations 4× on AutoDL.
- **Claude API translation:** Replace `translate.py` with an LLM call for more natural Simplified Chinese (literary idioms).
- **Image-to-video prompts:** Auto-generate prompt files alongside each illustration based on chapter text summary.
- **Resume support:** Skip already-processed folders if the pipeline crashes mid-run.
