## Fix Strategy

### Step 1: Normalize DPI before OCR

Never extract the raw embedded JPEG — render the page at a target DPI:

```python
import fitz  # pymupdf

def render_page_at_dpi(pdf_path: str, page_num: int, target_dpi: int = 300) -> Image:
    doc = fitz.open(pdf_path)
    page = doc[page_num]
    mat = fitz.Matrix(target_dpi / 72, target_dpi / 72)  # scale from 72pt base
    pix = page.get_pixmap(matrix=mat, alpha=False)
    return Image.frombytes("RGB", [pix.width, pix.height], pix.samples)
```

For 桃园结义 at target 300 DPI: renders 4082×2551 pts → ~17,000×10,630 px. That's huge. Better approach — **detect effective DPI and cap the upscale**:

```python
def render_page_normalized(doc, page_num: int, target_dpi: int = 300) -> Image:
    page = doc[page_num]
    # Get embedded image actual size
    img_list = page.get_images()
    if img_list:
        xref = img_list[0][0]
        base_img = doc.extract_image(xref)
        src_w, src_h = base_img["width"], base_img["height"]
        pt_w = float(page.rect.width)
        effective_dpi = src_w / (pt_w / 72)
        # Don't upscale beyond what the source supports (cap at source res)
        scale = min(target_dpi / 72, src_w / pt_w * 72 / 72)
        # But enforce minimum for OCR
        scale = max(scale, 150 / 72)
    else:
        scale = target_dpi / 72
    
    mat = fitz.Matrix(scale, scale)
    pix = page.get_pixmap(matrix=mat)
    return Image.frombytes("RGB", [pix.width, pix.height], pix.samples)
```

For 桃园结义, this will render the 1440×900 source — small but at least you're not wasting compute on upscaling artifacts.

### Step 2: Preprocess the low-DPI image

Since 桃园结义 is stuck at ~25 DPI source, you need super-resolution or at least aggressive upscaling + sharpening before OCR:

```python
import cv2
import numpy as np

def preprocess_for_ocr(img: np.ndarray) -> np.ndarray:
    # 1. Upscale 4x with Lanczos
    h, w = img.shape[:2]
    upscaled = cv2.resize(img, (w * 4, h * 4), interpolation=cv2.INTER_LANCZOS4)
    
    # 2. Sharpen
    kernel = np.array([[0,-1,0],[-1,5,-1],[0,-1,0]])
    sharpened = cv2.filter2D(upscaled, -1, kernel)
    
    # 3. Grayscale + adaptive threshold
    gray = cv2.cvtColor(sharpened, cv2.COLOR_RGB2GRAY)
    binary = cv2.adaptiveThreshold(
        gray, 255, cv2.ADAPTIVE_THRESH_GAUSSIAN_C,
        cv2.THRESH_BINARY, 31, 10
    )
    return binary
```

For best results at low DPI: consider **Real-ESRGAN** or **waifu2x** as a preprocessing step — they're specifically trained to upscale without blurring.

### Step 3: Replace hardcoded layout logic with dynamic region detection

```python
def detect_text_regions(img: np.ndarray):
    """Find text-containing bounding boxes via contour analysis."""
    gray = cv2.cvtColor(img, cv2.COLOR_RGB2GRAY) if len(img.shape) == 3 else img
    _, binary = cv2.threshold(gray, 0, 255, cv2.THRESH_BINARY_INV + cv2.THRESH_OTSU)
    
    # Dilate to merge nearby chars into text blocks
    kernel = cv2.getStructuringElement(cv2.MORPH_RECT, (20, 5))
    dilated = cv2.dilate(binary, kernel, iterations=2)
    
    contours, _ = cv2.findContours(dilated, cv2.RETR_EXTERNAL, cv2.CHAIN_APPROX_SIMPLE)
    regions = []
    for c in contours:
        x, y, w, h = cv2.boundingRect(c)
        area = w * h
        if area > 500:  # filter noise
            regions.append((x, y, w, h))
    return sorted(regions, key=lambda r: (r[0], r[1]))  # left-to-right, top-to-bottom
```

Or skip DIY layout analysis entirely and switch to **PaddleOCR** which handles Chinese layout detection natively:

## Bug 1: Rotation Direction Reverses Column Order (Primary)

This is why the sentences come out backwards. Look at the pattern across all iterations:

```
所以叫作黃巾軍...  ← sentence 2 (appears FIRST)
他們約定了一個日子...向官兵進攻  ← sentence 1 (appears LAST)
```

**What's happening geometrically:**

The text strip has two vertical columns. Traditional Chinese reads **right column → left column**. `ROTATE_270` (CW, PIL) maps:

```
Original pixel (x, y)  →  rotated (H-1-y, x)

Right column (x ≈ 300–500, any y)  →  new_y ≈ 300–500  (bottom half of rotated image)
Left column  (x ≈ 0–200,  any y)  →  new_y ≈ 0–200    (top half of rotated image)
```

So CW rotation puts the **left column at the top** of the rotated image. `_cluster_horizontal_lines` sorts rows **top→bottom** = reads left column first = **sentence 2 before sentence 1**.

**Fix: use ROTATE_90 (CCW) instead of ROTATE_270 (CW)**

```python

The problem is not just that the source is 25 DPI — **Step 1 actively degrades quality further** by rendering rather than extracting.

### What's happening in Step 1

```

### The fix for Step 1

Detect the "screen-capture-in-huge-page" pattern and bypass the renderer:

```python
import fitz
from PIL import Image
import io

def extract_page_image(doc: fitz.Document, page_num: int, 
                       target_long_side: int = 2700) -> Image.Image:
    """
    Smart extraction: use embedded JPEG if page is a low-DPI scan-in-large-container,
    otherwise render at a sensible DPI.
    """
    page = doc[page_num]
    img_list = page.get_images(full=True)
    
    if img_list:
        xref = img_list[0][0]
        base = doc.extract_image(xref)
        src_w, src_h = base["width"], base["height"]
        pt_w = page.rect.width
        effective_dpi = src_w / (pt_w / 72)
        
        if effective_dpi < 80:
            # Low-DPI-in-large-page: render would massively upscale garbage.
            # Extract native + upscale cleanly once.
            img = Image.open(io.BytesIO(base["image"])).convert("RGB")
            long_side = max(img.size)
            if long_side < target_long_side:
                scale = target_long_side / long_side
                nw = max(1, int(round(img.width * scale)))
                nh = max(1, int(round(img.height * scale)))
                img = img.resize((nw, nh), Image.Resampling.LANCZOS)
            return img
    
    # Normal PDF with real vector/high-DPI content: render at IMAGE_DPI
    dpi = 150  # sufficient for real content; 300 is overkill
    mat = fitz.Matrix(dpi / 72, dpi / 72)
    pix = page.get_pixmap(matrix=mat, alpha=False)
    return Image.frombytes("RGB", [pix.width, pix.height], pix.samples)
```
